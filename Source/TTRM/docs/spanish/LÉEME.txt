🧩 Torrent Tracker Registration Monitor (TTRM)

Autor: ElektroStudios
Website: https://github.com/ElektroStudios/Torrent-Tracker-Registration-Monitor
----------------------------------------------------------------------------------------------------------

TTRM es una herramienta diseñada para comprobar la 
disponibilidad de registro en múltiples trackers y 
foros de manera automatizada.

Permite analizar rápidamente el estado de registro 
mediante un sistema modular basado en plugins 
que facilitan la integración de nuevos sitios 
sin modificar el núcleo del programa.

ARCHIVO LÉEME
_____________

📋 Requisitos

	Este programa requiere de .NET Desktop Runtime 8.0 instalado en su equipo.

	Descargue el paquete de .NET Desktop Runtime 8.0 de 64-Bits aquí:
		► https://dotnet.microsoft.com/en-us/download/dotnet/8.0

📦 Arquitectura basada en plugins

	El programa funciona mediante un sistema de plugins (complementos).

	Cada plugin está compuesto por tres elementos principales:

		● Archivo JSON
		    Contiene los metadatos básicos del plugin: 
		    	nombre, descripción, URL de inicio de sesión o registro, etc.

		● Archivo de imagen PNG, JPG, BMP o ICO
		    Generalmente se trata de un banner o un favicon 
		    que representa visualmente al plugin.
		    Se recomienda que sea una imagen cuadrada (square), 
		    preferiblemente de 48×48 píxeles.

		● Archivo VB
		    Contiene el código fuente en VB.NET que implementa la 
		    lógica y el comportamiento del plugin dentro del programa.

📁 Ubicación de los plugins

	Todos los plugins se encuentran dentro de la carpeta "plugins".
	Si alguno no le resulta útil y quiere eliminarlo del programa, 
	puede borrar la carpeta correspondiente al nombre del plugin, 
	o moverlo a otro directorio fuera de su ubicación.

💡 Consejos

	1.  Algunos de los sitios web pueden mantener abierto el registro 
		durante varios días o incluso semanas y meses. Sin embargo 
		—y según la información recopilada y comentarios de usuarios— 
		otros sitios solo habilitan el registro durante períodos muy breves, 
		a veces menos de 24 horas, y entre ellos, sitios españoles como Torrentland.

		Por ese motivo, si no quiere perder la oportunidad de registrarse a tiempo, 
		se recomienda encarecidamente configurar la ejecución automática cada hora.
		De esta forma, TTRM podrá detectar con fiabilidad 
		periodos muy cortos de apertura de registro en un sitio web.

	2.  Si activa la éjecución en paralelo, los plugins se ejecutarán de cuatro en cuatro,
		significando abrir cuatro instancias del proceso chromedriver.exe (y sus respectivos 
		procesos hijos de Chrome.exe) al mismo tiempo. Esto acortará en gran medida el 
		tiempo total necesario para terminar de ejecutar todos los plugins, sin embargo, 
		también podría ocasionar problemas inesperados en la aplicación. Es experimental.
		
	3.  Si activa la éjecución automática de plugins, se recomienda seleccionar
		solamente aquellos plugins que realmente le interesen. A veces un sitio web
		puede estar caído o provocar un error inesperado que prolongue en exceso
		el tiempo total de ejecución.

	4.  Cada vez que ejecute un plugin, TTRM generará ciertos archivos en el disco 
		de forma automática. Esto es la caché de la aplicación. Si desea borrar la caché, 
		simplemente pulse el botón "Borrar caché" en las opciones del programa.

	5.  La primera vez que ejecute un plugin que interactúe con un sitio web protegido 
		por Cloudflare ("AvistaZ", "CinemaZ", "ExoticaZ", etc), puede fallar 
		el desafío de Cloudflare, sin embargo, las siguientes veces no debería fallar.
		Tenga en cuenta que, si borra la caché del programa, la próxima vez que
		ejecute el plugin será como si fuese la primera vez que lo hace. 

	6.  Si por cualquier motivo TTRM registrase errores extraños relacionados con Chrome
		al ejecutar un plugin, pruebe a cerrar todos los procesos que haya en ejecución 
		con los siguientes nombres: "Chrome.exe" y "ChromeDriver.exe". Esto lo puede hacer
		de manera sencilla con el siguiente comando en la CMD de Windows:
			Taskkill /F /T /IM "Chrome.exe" /IM "ChromeDriver.exe"
		Una vez que los procesos en ejecución hayan sido cerrados, 
		abra TTRM y vuelva a intentar ejecutar el plugin.
